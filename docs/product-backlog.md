# Product Backlog

| Backlog NO. | 메인 메뉴 | Sub-1 메뉴 | Demo Scenario |
|---|---|---|---|
| 1 | 홈 | 교수 회원가입 | 1. Chrome을 연다.<br>2. Chrome에서 "https://localhost/signup"을 입력한다.<br>3. 회원가입 페이지에 '아이디', '비밀번호', '이름', '소속'을 입력한다.<br>4. 가입하기 버튼을 클릭한다.<br>5. 가입 완료 메시지가 출력되며 로그인 페이지로 이동한다. |
| 2 | 홈 | 교수 로그인 | 1. Chrome에서 "https://localhost/login"을 입력한다.<br>2. 로그인 페이지에 '아이디', '비밀번호'를 입력한다.<br>3. 로그인 버튼을 클릭한다.<br>4. 교수 메인 대시보드 페이지로 이동한다. |
| 3 | 교수 대시보드 | 시험 목록 조회 | 1. 교수 계정으로 로그인한 상태에서 메인 대시보드에 진입한다.<br>2. 자신이 개설한 시험 목록이 시험명, 시험 코드, 시작/종료 시각, 명단 인원, 응시자 수와 함께 표시된다.<br>3. 시험이 없을 경우 "개설된 시험이 없습니다" 안내가 표시된다.<br>4. 우측 상단의 '새 시험 만들기' 버튼이 표시된다. |
| 4 | 교수 대시보드 | 시험 개설 | 1. 교수 대시보드에서 '새 시험 만들기' 버튼을 클릭한다.<br>2. 시험 개설 페이지로 이동한다.<br>3. '시험명', '시작 시각', '종료 시각'을 입력한다.<br>4. '문항 추가' 버튼을 클릭하여 문항 유형(주관식/객관식), 문항 본문, 배점을 입력한다.<br>5. 주관식 선택 시: 정답(참조용)을 입력한다.<br>6. 객관식 선택 시: 선택지를 N개 입력하고 정답 선택지에 체크한다 (정답을 여러 개 체크할 수 있다).<br>7. 필요 시 문항에 이미지를 첨부한다 (드래그 앤 드롭 또는 파일 선택).<br>8. 문항을 N개 등록한 후 '응시자 명단 등록' 버튼을 클릭한다.<br>9. 응시자 명단 등록 페이지로 이동한다. |
| 5 | 교수 대시보드 | 응시자 명단 관리 | 1. 응시자 명단 등록 페이지 상단에서 '개별 추가' 또는 '일괄 추가' 탭 중 하나를 선택한다.<br>2. '개별 추가' 탭 선택 시: '학번 입력 칸'과 '이름 입력 칸'에 정보를 직접 타이핑한 후 '추가' 버튼을 누른다.<br>3. '일괄 추가' 탭 선택 시: 복사한 학번·이름 텍스트 뭉치를 '일괄 등록 텍스트 창'에 붙여넣고 '명단 추출' 버튼을 누른다.<br>4. 화면 하단의 '등록 예정자 미리보기 목록'에서 추가된 학생들의 정보를 최종 확인한다 (필요 시 목록에서 개별 삭제 가능).<br>5. '시험 개설 완료' 버튼을 클릭한다.<br>6. 시험 상세 페이지로 이동하며, 6자리 시험 코드가 발급되어 표시된다. |
| 6 | 교수 대시보드 | 시험 상세 조회 | 1. 교수 대시보드의 시험 목록에서 시험 한 건을 클릭한다.<br>2. 시험 상세 페이지로 이동한다.<br>3. 시험 정보(시험명, 시간, 코드), 문항 목록(정답 포함), 응시자 명단, 응시 학생 목록(상태, 총점 포함)이 표시된다.<br>4. '감독관 링크 복사' 버튼을 클릭하면 감독관용 링크가 클립보드에 복사된다. |
| 7 | 응시 | 시험 코드 입력 | 1. Chrome에서 "https://localhost/exam"을 입력한다.<br>2. 시험 코드 입력 페이지에 6자리 시험 코드를 입력한다.<br>3. '입장' 버튼을 클릭한다.<br>4. 학번/이름 입력 페이지로 이동한다. |
| 8 | 응시 | 학번/이름 입력 후 응시 시작 | 1. 학번/이름 입력 페이지에서 '학번', '이름'을 입력한다.<br>2. '시험 시작' 버튼을 클릭한다.<br>3. 시험 시작 시각 이전이면 "시험이 아직 시작되지 않았습니다 (시작 시각: HH:MM)" 안내가 표시되고 시험은 시작되지 않는다.<br>4. 시험 종료 시각 이후이면 "시험이 종료되었습니다" 안내가 표시되고 시험은 시작되지 않는다.<br>5. 입력한 학번과 이름이 응시자 명단에 등록된 정보와 정확히 일치하면 브라우저가 전체 화면 모드로 전환된다.<br>6. 전체 화면 진입이 확인되면 시험 응시 화면으로 이동한다.<br>7. 시험명, 첫 번째 문항, 답안 에디터, 남은 시간이 표시된다.<br>8. 명단에 없거나 이름이 일치하지 않으면 "응시 권한이 없습니다" 안내가 표시되고 시험은 시작되지 않는다.<br>9. 전체 화면 진입에 실패하면 "전체 화면 모드가 필요합니다" 안내가 표시되고 시험은 시작되지 않는다. |
| 9 | 응시 | 답안 작성 및 문항 이동 | 1. 시험 응시 화면에서 첫 번째 문항이 표시된다. 문항에 이미지가 첨부되어 있으면 함께 표시된다.<br>2. 주관식 문항이면 답안 에디터에 텍스트를 입력한다.<br>3. 객관식 문항이면 선택지를 클릭하여 선택한다. 여러 선택지를 동시에 선택할 수 있고 선택을 변경할 수 있다.<br>4. '다음 문항' 또는 '이전 문항' 버튼을 클릭하여 문항을 이동하면 작성한 답안이 그대로 유지된다. |
| 10 | 응시 | 답안 제출 | 1. 시험 응시 화면에서 '제출' 버튼을 클릭한다.<br>2. "제출하시겠습니까?" 확인 다이얼로그가 표시된다.<br>3. '확인' 버튼을 클릭한다.<br>4. 객관식 문항은 정답을 모두 정확히 맞춘 경우에만 만점, 그 외에는 0점으로 자동 채점된다.<br>5. 제출 완료 페이지가 출력된다. |
| 11 | 교수 대시보드 | 주관식 채점 | 1. 시험 상세 페이지의 응시 학생 목록에서 학생 한 명을 클릭한다.<br>2. 학생 답안 상세 페이지로 이동한다.<br>3. 객관식 문항은 자동 채점된 점수가 표시된다.<br>4. 주관식 문항은 학생 답안과 함께 정답(참조용), 점수 입력란이 표시된다.<br>5. 점수 입력란에 0 이상 배점 이하의 점수를 입력한다.<br>6. '저장' 버튼을 클릭하면 점수가 저장되고 학생 총점이 자동 갱신된다. |
| 12 | 응시 | 응시 환경 이탈 처리 | 1. 시험 응시 중 학생이 알트탭으로 다른 창으로 전환하거나, ESC로 전체 화면을 해제하거나, 브라우저를 최소화한다.<br>2. 응시 화면이 흐림 처리되고 경고 오버레이가 표시된다.<br>3. 학생 화면 상단에 누적 이탈 횟수가 갱신된다.<br>4. '시험으로 돌아가기' 버튼을 클릭하면 전체 화면으로 재진입하고 응시가 재개된다.<br>5. 이탈 시작 시각과 복귀 시각이 모두 기록되어, 사후 검토 시 이탈 지속 시간을 확인할 수 있다. |
| 13 | 시스템 | 실시간 이벤트 수집 | 1. 학생이 응시 화면에 진입하면 클라이언트가 서버와 실시간 통신 채널을 수립한다.<br>2. 학생이 답안을 작성하는 동안 다음 행위가 즉시 서버에 전송된다: 붙여넣기, 화면 이탈/복귀, 전체화면 해제, 캡처 단축키 입력.<br>3. 이탈과 복귀는 쌍으로 묶여 지속 시간이 함께 기록된다.<br>4. 서버는 받은 이벤트를 즉시 처리하여 감독관 화면 갱신과 주목도 점수 계산에 사용한다. |
| 14 | 시스템 | 추가 이벤트 수집 | 1. 학생이 응시하는 동안 키 입력, 객관식 선택 변경, 문항 간 이동 정보가 클라이언트에 누적된다.<br>2. 1분 간격으로 누적된 데이터와 현재 작성 중인 답안 스냅샷이 서버에 전송되어 저장된다.<br>3. 답안 제출 시 마지막 1분 미만의 잔여 데이터가 함께 전송된다.<br>4. 저장된 데이터는 사후 답안 재생에 사용된다.<br>5. 답안 스냅샷은 네트워크 단절 등 비정상 상황 시 시간 만료 자동 제출의 백업 데이터로 활용된다. |
| 15 | 교수 대시보드 | 종료된 시험 답안 재생 | 1. 종료된 시험의 상세 페이지에서 응시 학생의 '재생' 버튼을 클릭한다.<br>2. 답안 재생 페이지로 이동한다.<br>3. 문항 선택 드롭다운에서 문항을 선택한다.<br>4. 주관식 문항: '재생' 버튼 클릭 시 학생이 작성한 순서대로 답안이 한 글자씩 채워진다. 붙여넣기는 한 번에 표시된다.<br>5. 객관식 문항: 학생의 선택 변경 이력이 시간순으로 표시된다 (예: "1번 선택 → 1번, 3번 선택 → 2번으로 변경").<br>6. 학생이 다른 문항으로 이동했다가 돌아온 구간은 타임라인에 회색 빈 구간으로 표시된다.<br>7. 학생이 화면을 이탈한 구간은 타임라인에 빨간색 막대와 지속 시간으로 표시된다.<br>8. 화면 이탈 직후 객관식 답을 변경한 경우 해당 변경이 빨간색으로 강조된다.<br>9. 배속 컨트롤(1배, 2배, 5배, 10배)로 재생 속도를 변경할 수 있다.<br>10. 타임라인 바를 클릭하여 임의 시점으로 이동할 수 있다. |
| 16 | 감독관 | 감독관 대시보드 접속 | 1. 교수가 시험 상세 페이지에서 '감독관 링크 복사' 버튼을 누른다.<br>2. 복사한 감독관 링크를 다른 브라우저(또는 다른 노트북)에 붙여넣는다.<br>3. 별도 로그인 화면 없이 바로 감독관 대시보드에 진입한다 (링크에 포함된 토큰으로 자동 인증).<br>4. 시험명, 시작/종료 시각, 응시자 수가 상단에 표시된다.<br>5. 응시 중인 학생 카드 목록이 표시된다 (학번, 이름, 현재 진행 문항, 마지막 활동 시각). |
| 17 | 감독관 | 주목도 순 정렬 + 학생 상세 | 1. 감독관 대시보드의 학생 카드가 주목도 점수 높은 순으로 정렬된다.<br>2. 점수 범주별 색상으로 강조된다 (강조/주의/보통/평범).<br>3. 학생이 의심스러운 행동(붙여넣기, 화면 이탈 등)을 하면 1초 이내에 카드 색상과 위치가 갱신된다.<br>4. 학생 카드 클릭 시 우측 상세 패널이 열린다.<br>5. 패널에 시그널별 발생 횟수(이탈 N회, 평균 이탈 지속 시간 X초, 붙여넣기 N회 등), 최근 발생한 시그널 목록, 작성 중인 답안 미리보기가 표시된다. |
| 18 | 감독관 | 이벤트 피드 | 1. 감독관 대시보드 한 영역에 실시간 이벤트 피드가 표시된다.<br>2. 시험 전체에서 발생한 주요 이벤트가 시간 역순으로 표시된다.<br>3. 각 행에 학번, 이벤트 종류, 어느 문항에서 일어났는지, 발생 시각이 표시된다.<br>4. 화면 이탈과 복귀는 묶여서 지속 시간과 함께 한 줄로 표시된다.<br>5. 새 이벤트 발생 시 피드 상단에 즉시 추가된다. |
| 19 | 감독관 | 확인함 처리 | 1. 학생 상세 패널에서 '확인함' 버튼을 클릭한다.<br>2. 확인 다이얼로그에서 메모를 입력한다.<br>3. '확인' 버튼 클릭 시 해당 학생의 주목도 점수가 일시 감쇠된다.<br>4. 카드가 상단에서 내려가고 다른 학생이 상위로 올라온다.<br>5. 처리 기록이 저장된다. |
